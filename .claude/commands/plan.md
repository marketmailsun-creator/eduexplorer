---
description: Generate a comprehensive project plan as a markdown document
allowed-tools: Bash(find:*), Bash(cat:*), Bash(git:*)
---

Analyze the entire project structure, including all source files, configuration, tests, and documentation. Then create a detailed markdown plan document at `docs/PLAN.md` that includes:

1. **Project Overview**: High-level summary of what the project does
2. **Architecture**: Key components, modules, and how they interact
3. **Current State Assessment**: What exists, what works, what needs improvement
4. **Technology Stack**: Languages, frameworks, libraries, and tools in use
5. **Dependency Map**: Key dependencies and their purposes
6. **Directory Structure**: Annotated tree of the project layout
7. **Branch Overview**: Current branches and their purposes

Format using proper markdown with headers, tables, code blocks, and bullet points.
